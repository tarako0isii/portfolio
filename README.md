## 日本人向けインドレビューアプリ「侍インドレビュー」

**概要**  
チームで開発した、日本人向けのインドレビューアプリです。
本アプリでは、新規登録時に「辛さ耐性度」「清潔重視度」「快適重視度」「混雑苦手度」の
4項目を1〜5段階で設定します。
設定されたユーザーの感性をもとに、飲食店・観光地・ホテルなどが自動で表示され、
自分に合った場所を探しやすくなる仕組みです。

また、レビュー投稿時にも同じ4項目で評価を行い、
日本人ユーザー同士で感性に基づいたレビューを共有することができます。
海外での旅行においても、日本人の感覚に近い情報をもとに、
安心して行動できることを目的としたサービスです。

本プロジェクトでは、主にバックエンドの実装を担当しました。

**制作背景**  
インドで現地調査を行った際、飲食店や観光地、ホテルなどを選ぶ場面で、
多くのレビューを参考にしました。
自動翻訳を使えば海外のレビューも読むことはできますが、
文化の違いにより評価の基準が分かりづらいと感じることが多くありました。

特に飲食店では、「辛くない」と書かれていたにもかかわらず、
実際には非常に辛くて食べられないという経験をしました。
この体験から、日本人の感性で評価されたレビューサービスが不足していると感じ、
飲食店や観光地、ホテルなどを日本人の基準で評価できる
レビューアプリの制作を考えました。

**使用技術**  
- バックエンド：Python  
- フロントエンド：React  
- データ通信：JSON  
- データベース：
  - SQLite（開発環境）
  - PostgreSQL（Neon / 本番環境）
- 使用API : GoogleMapAPI / GooglePlacesAPI
- その他：Git / GitHub / Vercel 


**工夫した点・苦労した点**  
バックエンドでは、Pythonで取得したデータをJSON形式でレスポンスし、
React側で扱えるように実装しました。
また、チーム開発としてGitHubのブランチ機能を用い、
役割分担をしながら開発を進めた点も大きな学びでした。

**今後の改善点**  
ユーザーインターフェースが未完成の部分があるため、
操作性や視認性の改善が必要だと考えています。
また、現在設定している4項目が本当に日本人の感性に適しているのかについて、
論理的な根拠を持って説明できるよう、指標の再検討を行いたいです。
加えて、マップ上に表示されるピンの違いがユーザーに分かりづらいため、
視覚的に区別できる工夫を行いたいと考えています。
